3.2 图像变换

3.2.1 图像的缩放

  • resize(src, dst, dsize, fx, fy, interpolation)
  • fx：x轴的缩放因子；fy: y轴的缩放因子
  • interpolation：插值算法
    • INTER_NEAREST，邻近插值，速度快，效果差
    • INTER_LINEAR，双线性插值，原图中的4个点（默认）
    • INTER_CUBIC，三次插值，原图中的16个点
    • INTER_AREA，效果最好
  • 如果dsize没有设定，那么就会选择fx和fy的值

3.2.2 图像翻转

  • flip(img, flipCode)
  • flipCode == 0，上下
  • flipCode > 0，左右
  • flipCode < 0，上下+左右

3.2.3 图像旋转

  • rotate(img, rotateCode)
  • ROTATE_90_CLOCKWISE
  • ROTATE_180
  • ROTATE_90_COUNTERCLOCKWISE

3.2.4 图像的仿射变换

  • 仿射变换是图像旋转、缩放、平移的总称
  • warpAffine(src, M, dsize, flags, mode, value)
    • M:变换矩阵
    • dsize输出尺寸大小
    • flag：与resize中的插值算法一致
    • mode：边界外推法标志
    • value：填充边界的值
  • 平移矩阵：矩阵中的每个像素由（x，y）组成，变换矩阵是2x2的矩阵，平移向量为2x1的向量，所以平移矩阵为2x3矩阵
  • 变换矩阵
    • getRotationMatrix2D(center, angle, scale)
      • center中心点，angle角度，scale缩放比例
    • getAffineTransform(src[], dst[])
      • 通过三个点可以确定变换的位置

3.2.5 透视变换

  • warpPerspective(img, M, dsize, …)
    • M是变换矩阵，dsize是目标图像大小
  • 变换矩阵：getPerspectiveTransform(src, dst)
    • 四个点（图形的四个角）

3.3 图像滤波

  • 一幅图像通过滤波器得到另一幅图像；其中滤波器又称为卷积核，滤波的过程称为卷积

3.3.1 卷积基础（卷积核、锚点、边界扩充、步长、低通/高通滤波）

  • 卷积过程：
  • 卷积核的大小：一般为奇数，如3x3、5x5、7x7等；一方面是增加padding的原因；另一方面是保证锚点在中间，防止位置发生偏移
  • 锚点：
  • 边界扩充：当卷积核大于1且不进行边界扩充，输出尺寸将相应缩小；当卷积核以标准方式进行边界扩充，则输出数据的空间尺寸将与输入相等
    • 计算公式：N = (W - F + 2P)/S + 1
      • N输出图像大小，W源图大小，F卷积核大小，P扩充尺寸，S步长大小
  • 步长：
  • 低通滤波与高通滤波：低通滤波可以去除噪音或平滑图像；高通滤波可以帮助查找图像的边缘
  • 图像卷积：filter2D(src, ddepth, kernel, anchor, delta, borderType)
    • ddepth：一般设置成-1，保持输入输出位深一致
    • kernel：不同的卷积效果不同
    • anchor：锚点，不设置时根据kernel尺寸自动确定
    • delta：每次卷积后加的偏移值
    • borderType：边界类型
  • 例子：一个5x5的核，每个值乘以1/25做平均值，可使图像平滑

3.3.2 低通滤波

3.3.2.1 方盒滤波与均值滤波

3.3.3 高通滤波

3.3.3.1 Sobel算子
  • 先向x方向求导，再向y方向求导，最终结果|G|=|Gx|+|Gy|
  • API：Sobel(src, ddepth, dx, dy, ksize=3, …)
    • ksize设置成-1即为Scharr算法
3.3.3.2 Scharr算子
  • 与Sobel类似，但kernel固定为3x3，可识别更细小的线条
  • 只能求单方向边缘（dx,dy不能同时为1）
  • API：Scharr(src, ddepth, dx, dy, …)
3.3.3.3 Laplacian算子
  • 可同时求两个方向的边缘，但对噪音敏感，需先降噪
  • API：Laplacian(img, ddepth, ksize=1, …)

3.6 图像轮廓

3.6.1 轮廓基础概念

  • 具有相同颜色或强度的连续点的曲线
  • 可以不用灰度直接查找轮廓
  • 示例：
  • 作用：图形分析、物体识别与检测
  • 注意：为了检测准确性，需先二值化或Canny；画轮廓时会修改输入图像

3.6.2 轮廓查找（findContours）

  • API：findContours(img, mode, ApproximationMode, …)
  • 返回值：contours（轮廓列表），hierarchy（层级关系）
  • mode（查找模式）：
    • RETR_EXTERNAL=0：只检测外轮廓
    • RETR_LIST=1：检测的轮廓不建立等级关系，顺序从里到外、从小到大
    • RETR_CCOMP=2：每层最多两级
    • RETR_TREE=3：按树形存储轮廓（常用），从大到小、从右到左
  • ApproximationMode（轮廓近似方法）：
    • CHAIN_APPROX_NONE：保存所有轮廓上的点
    • CHAIN_APPROX_SIMPLE：只保存角点

3.6.3 绘制轮廓（drawContours）

  • API：drawContours(img, contours, contourIdx, color, thickness, …)
    • contourIdx = -1 表示绘制所有轮廓
    • color：例如(0,0,255)
    • thickness：线宽，-1表示填充

3.6.4 轮廓的面积与周长

  • 面积：contourArea(contour)
  • 周长：arcLength(curve, closed) （closed表示是否闭合）

3.7.3 GrabCut法

3.7.3.1 原理（交互式前景提取）
  • 用户指定前景大体区域，剩余为背景；可明确指定某些地方为前景或背景
  • 采用分段迭代分析前景物体形成模型树，最后根据权重决定像素类别
3.7.3.2 grabCut API 与鼠标事件
  • API：grabCut(img, mask, rect, bgdModel, fgdModel, iterCount, mode)
    • mask值：BGD=0背景，FGD=1前景，PR_BGD=2可能背景，PR_FGD=3可能前景
    • bgdModel/fgdModel：np.float64类型的(1,65)零数组
    • mode：GC_INIT_WITH_RECT 或 GC_INIT_WITH_MASK
  • 实战：主体结构 + 鼠标事件处理

3.7.4 MeanShift分割（色彩平滑滤波）

  • 不是严格的分割，而是色彩层面的平滑滤波
  • 中和色彩相近的颜色，平滑细节，侵蚀小面积颜色区域
  • 以任一点P为圆心，半径sp，色彩幅值sr进行迭代
  • API：pyrMeanShiftFiltering(img, sp, sr, maxLevel=1, termcrit=…)

3.8.1 视频背景处理概念

  • createBackgroundSubtractorMOG2(history=500, …, detectShadows=True)

3.8.4 GMG去背景（抗噪性更强）

  • 静态背景图像估计和每个像素的贝叶斯分割
  • createBackgroundSubtractorGMG(initializationFrames=120)
